# A02 – Git, Vscode, and GitHub Tutorial

# PART 1: Directions on Using Git, Vscode, and GitHub

## Part 1/Step 1: Install Git
1. Go to the official Git website: <https://git-scm.com/> and download the installer for your operating system (windows or Mac).
2. Install: Run the setup and keep the default settings.
3. Verify: Open your Terminal/Command Prompt and type git --version to ensure it's ready.

# Part 1/Step 2: Set Up your editor - Tutorial if you choose to use VS Code
1. Download the installer from <https://code.visualstudio.com/>.
2. Create a folder on your Desktop named A02
3. Launch Vs Code, go to File > Open Folder, and select your A02 folder.
4. Look for the Source Control icon on the left sidebar. This is where VS Code will track all Git changes for this specific folder.

## Part 1/Step 3: Create GitHub Repository called A02
1. Go to <https://github.com>
2. Once logged in, click the "+" icon in the top right corner and select “New repository”.
3. Name the repository A02.
4. Select Public.
5. Check “Add a README file”.
6. Click “Create repository”.

## Part 1/Step 4: The Workflow 
1. Copy your repository URL from GitHub.
2. In VS Code, press Ctrl+Shift+P (Windows) or Cmd+Shift+P (Mac), type "Git: Clone," and paste the URL to link your local folder to GitHub.
3. Make necessary changes to your README file.

## Part 1/Step 5: Make Your First Commit
1. In the Source Control tab, click the + icon next to file.
2. Enter a clear commit message such as: "Task: Creating Repository A02"
3. Click the Commit button.

## Part 1/Step 6: Push Changes to GitHub
1. Click the Sync Changes or Push button in VS Code.
2. Refresh GitHub in your browser to confirm the changes appear online.

# PART 2: GLOSSARY OF TERMS
1. **Branch** - A separate version of a project used to develop features independently from the main code.
2. **Clone** - A copy of a repository downloaded from GitHub to a local computer.
3. **Commit** - A saved snapshot of changes made to files in a repository.
4. **Fetch** - A command that retrieves updates from a remote repository without merging them.
5. **GIT** - A distributed version control system that tracks changes in source code.
6. **Github** - A web-based platform that hosts Git repositories for collaboration and version control.
7. **Merge** - The process of combining changes from one branch into another.
8. **Merge Conflict** – A situation that occurs when Git cannot automatically resolve differences between two versions of a file.
9. **Push** - A command used to upload local commits to a remote repository on GitHub.
10. **Pull** - A command that retrieves and merges changes from a remote repository into a local repository.
11. **Remote** - A version of a repository hosted on a server such as GitHub.
12. **Repository** - A storage location for a project that contains files, folders, and version history managed by Git.
